## Documentation Standards

### Document Naming
- Use UPPERCASE for root-level important docs (README.md, CHANGELOG.md)
- Use PascalCase or Title Case for docs/ subdirectory
- Use hyphens for multi-word files (OPERATIONS-RUNBOOK.md)
- Use .md extension for all markdown files

### Document Structure
All documentation should include:
1. **Title and metadata** (project, version, date)
2. **Table of contents** (for docs >500 lines)
3. **Clear sections** with headers
4. **Code examples** where applicable
5. **Cross-references** to related docs
6. **Update history** (version, date, author)

### Document Maintenance
- Review quarterly
- Update version numbers
- Archive outdated documentation
- Keep examples current
- Verify links work
